BB 9300  BB 9900  Lumia 710  HTC Desire C  Nokia 101  Nokia 5800

曼游记之英国南海岸

6月的欧洲之行想要写的太多，尽管留了不少笔记，但是一直没有合适的心境来整理。所以曼游记首次打破时间顺序，先写这次英国南海岸的四天休闲之旅。

这次出行主要是想最后感受一下夏天，去年就想去怀特岛，今年算是实现了。总的来说玩得比较轻松，因为不像伦敦那样景点繁多，就是海边吹吹风，晒晒太阳。岛上基本都是退休老人和前来度假的家庭。因为晚上没有什么娱乐活动，所以笔记比较详细，懒得整理，直接同步加复制了。

 

2013-08-31 阴有晴

9月即将来临，凛冬将至。赶紧出发感受一下夏天的余温。
此行主要目的地是怀特岛，因为布莱顿离得不远，所以就顺便游览一番。这里有着英村最受欢迎的海滩。
作为刚从法国蔚蓝海岸回来的我，为什么又去海边呢? 因为这里有著名的自(luo)然(ti)海滩。
由于火车奇贵，只有坐大巴了，5个小时坐得真是难受。其实车的条件还不错，座椅宽敞，有靠头的部分，就是忽冷忽热的空调不知道是闹哪样。
途中还经过一个休息站，两年前去威尔士经过过，咖啡还是坑爹的贵。
伦敦市区游客如织，堵车严重，将近一个小时才从市郊开到车站。稍作休息便搭上火车来到了布莱顿。
如同大多数的英村城市，布莱顿面积也不大，主要就是两条街。从火车站出来一路向南，印象并不深。终于踏上了海滩，这绝对是英村最…磨鞋的沙滩之一，全是鹅卵石。可能是来得晚了，沙滩上人不多，可能还是鸟多。
犹豫了一下还是没有去自然海滩，一来凉风习习应该没有什么人，二来据说这个海滩好多基。
布莱顿也有个摩天轮，边上就是布莱顿码头。其实就是娱乐场，虽然没有去玩，不过看着人们开心的场景，心情也随之好了不少。坐在码头的木桌前，吃着当地著名的fish&chips，任由海风肆虐我为数不多的头发，太TM的浪漫了。
吃完就听妈妈的话，天黑前回了旅舍。这旅舍的木质楼梯吱呀作响，和窗外的海鸥倒是合成了不错的重奏。稍感失望，略觉疲惫，便在这双重奏中早早沉睡。
2013-09-01  晴
早上起床 看到邻床下铺睡着一个身着三点的金发性感美女，然后发现边上还躺着一个人，结果还是一个美女。。不愧是最大的基村。
坐火车前往普斯茅斯。提前一站在市中心下车，发现市区没有什么好看的。来到海港，非常热闹，建筑很现代，还有大型商场，有点像利物浦。
中午坐上轮渡前往，坐在上层阳光甲板，晒太阳吹海风。大约半小时就到了怀特岛，坐上了红皮车。三节车厢并不联通，一半是木质的，开起来摇摇晃晃，也许正因为此，座位是超高软座。坐在弹簧上穿越丛林，和蒸汽火车相交，最终来到我的住地，shanklin. 这个小镇有一些古老的建筑，是茅草顶，很有特色。在邮局买邮票时还碰到老奶奶店员送现烤蛋挞，原来是这家超市今晚开始就关门装修了。几经周折，来到了预订的酒店。酒店离海岸不远，装修有点上世纪的风格。
休整片刻，便坐汽车前往sandown, 这里的海滩是细沙滩。大多数都是家庭度假，还有不少老年人，做沙雕的，冲浪的，划艇的，游泳的，晒太阳的，显得很休闲。我走了一段，忍不住买了双沙滩鞋，卷起裤脚就下水了。这几年海滩去过不少，这还是第一次触水。
沿着海岸线走回shanklin, 一边是海，一边是断崖。不少家庭在海边租了小屋来聚会。海滩上还有不少狗，和主人玩得很欢，不少还勇敢的冲进海水接球或者追逐海鸥。
日落前在海边美餐一顿，回到酒店。
2013-09-02 晴
早上起来已经九点半了，酒店有早餐提供，英式早餐加自助，吃饱上路。找车站走了不少冤枉路，最后坐上饶远路的3路，一个小时抵达Newport. 换乘7路，先是去了castle. 从小镇穿过山丘和小溪，来到carisbrook城堡。城堡很有历史，曾经是岛上最强的防御工事。查理一世曾被关押于此，期间还试图越狱，最终难逃被砍头的结局。维多利亚女王的小女儿也曾把这里作为官邸。我更喜欢冷兵器时代，依据地势修建防御，讲究战术战法，刀光剑影，热血激昂。
接着继续前往the needles.坐缆车下悬崖，缆车只有一块木板，一个架子，摇摇晃晃。海滩边的悬崖颜色炫丽，远处就能看到白色薄片岩石。坐上快艇，绕着needles转了一圈。好久没有在海上坐船了，所幸今天海浪不大。
坐上汽车返回，无接缝转乘，回到shanklin, 因为周日没什么人，商店也关的早，就去买了点薯条，坐在海边当晚饭吃。
2013-09-03 晴
早上起来坐车经newport到Osborne house. 这里是维多利亚女王最爱的行宫，最后在此逝世。坐马车从入口坐到House。马车载客九人，超载一人。。车上有个老太太说就像当年女王一样坐马车回宫。很快就到达了行宫，黄色的外表并不是很突出，整个房子也不算大。走进屋子，先是一些雕塑和油画的陈列。比较有意思的是皇家婴儿室，有婴儿床，边上还有不少玩具。女王卧室不算大，这里也是当年她去世的地方。位置最好的是女王的书房，这里有三面落地窗，可以远眺大海。女王多次在日记里赞美这里的风景，描述她和家人的温馨的家庭生活。
走出House 便是装点别致的后花园。从后花园出发，穿过林荫道和草地，便可以到达海边。中间路过一个贮冰室，隐藏在一片小树林中。小屋外观像一个墓穴，边上的丛林里时而传出窸窣声，入口通道非常黑暗，走进去不禁打一个冷颤，尽头是一个很大的天井，应该就是储冰的地方了。来到海边，可以远眺对岸的英国大陆。岸边有女王的半拱形座椅，还有一个戏水车。因为女王不能在公众面前穿着暴露，所以把这个车推进海水里，女王在这个封闭的车里娱乐。在树林间还有一个瑞士小屋，曾用来给皇子皇孙学习和娱乐。海滩，树林，草原，宜人的气候，怪不得女王喜爱这里了。
由于参观时间比预计久，出来后就直奔海港。刚好赶上轮渡。轮渡和来时不同，是大型载车的，因此速度不快，一个小时后才抵达南安普顿。没有细看这个城市，匆匆一瞥，感觉没有什么特色。由于时间紧迫，没有时间吃饭，在车站买了个鸡肉卷便上火车了，然后在伯明翰转车回曼彻斯特。转车时由于晚点，差点没赶上车，前往曼城的车也晚点，但是居然准时发车，幸亏小跑了两步，不然可就惨了。

Macbook Air 11

使用macbook air 11已经快两个月了，已经是我的常用设备，不写进玩转数码显得不够意思。

之前从来没有想过要买苹果的macbook，一来价格昂贵，二来系统不熟悉，没有常用的软件（游戏）。但从去年开始学习编程之后，发现win 系统实在不够友好，因此在raspberry pi 上接触了linux系统。后来为了学习ruby，用virtual box虚拟了ubuntu。期间受到了多篇各路大神关于macbook一步到位的文章的毒害。

因为过去一年里攒了一点钱，加上需要使用keynote, pages等应用，未来开发web app的可能性，于是下狠心买了。英村坑爹的价格也没有阻止我买macbook的决心。后果就是，银行账户基本空了。

为什么买air而不是pro？买11而不是13？

的確，pro的性能更加强劲，屏幕清晰，升级便利。选择air的原因很简单：便携。

Air 11 基本上和我的ipad 2差不多，放在包里不觉得重。不像我的小黑，基本钉死在家里的书桌上了。

今年6月推出的新air续航能力强劲，理论可达9小时。实际使用在6小时左右，应付一天的工作量，绰绰有余。

Mac OS 上手问题

之前完全没有接触过，用过ios和ubuntu多少对mac os的使用提供了些基础。我比较反感买mac装win的人，其实mac os在很多方面都很便捷。触摸板和鼠标操作上手之后就非常方便，以至于我在用小黑时也下意识地用那些手势（当然无效）。虽然以前常见的软件找不到了，破解应用很难找，但从另一个角度来说也是优势，下面会谈到。

Macbook 效率

不少文章都提到过这点，的確，使用macbook，很少会分心。因为没有游戏，没有QQ，没有影音。我给它的定位就是工作机。

祭旗★欧洲申根签证不完全指南

预谋了好几年的欧洲行终于要拉开序幕了。

一大早收到了申根签证，好久没有这么开心了。因为递交材料的时候有几个问题，加上持psw递签的RP论，当时已经做好拒签准备，谁想否极泰来，三天后就收到了这份儿童节大礼。最后给的是一个月多次往返的签。

很少写这种攻略指南类，权当还愿，仅供参考。

所需材料（意签）：

网上很多文章(其实也就是抄来抄去那几篇)详细介绍的，有一些会误导或者失效，因此我强烈建议自己去官网看Document Required。重点说明几点。

1.往返机票。（官网要求fully paid return tickets and  proof of payments must be clearly shown。我的返程机票出了问题，机票确认信上没有支付信息，工作人员要求我补这个材料，我说实在没有，然后就被注明缺少材料，擦。这里强烈鄙视jet2。）

2.国际国内交通。（国内的交通证明是不需要的，但是跨国之间的证明最好有。我用的是interrail的通票，在欧洲住满6个月的人都可以买，26岁一下更便宜，嘿嘿。我是递签的前一天才买，当时怕缺这个有影响。递完之后很后悔，万一拒签，这200多磅就废了。）

3.学校信/雇主信。（在校的好办，但是要注意抬头要写The Consulate General of Italy, London，要有真人签名；我是厚颜无耻地问老板要了抬头纸，然后自己写了假条…嘘…）

4.酒店证明。（用booking.com, hostelworld, ibis 等可以取消预订的都行，但是一定要包含行程的每一天。工作人员是一天接一天查的，不要有gap）

5.资金证明。(银行账单，我的最新账单是到递签前26天的，结果工作人员说不行，要两周以内的，给了我一个小时去补材料。我冒雨前往银行拉了一张没有地址的单，和有地址的单一起交上去了。saving account是不认的。存款当然是越多越好，需要包含你的住宿费加上30磅/天的生活费，工作人员会算出来填到一张表上的。)

6.护照、有效签证(至少还有三个月)、照片、保险。

7.工作人员说是最少七个工作日，实际三个工作日。即便如此，旺季递签还是提早预约的好，万一要缺材料或者被拒，还有机会弥补。

8.上述所谓工作人员都是阿叉…非常的…不说了。

Start Using Heroku and Problem Resolve

1. Signup

2.Download and install Heroku:

toolbelt.heroku.com

Login:

$ heroku login

3.Create heroku remote

$ heroku create

$ git remote -v

$ git remote rm heroku

$ git remote add heroku git@heroku.com:xxxx.git     #Here, xxxx.git is the git file created in the first line of this step

4. Deploying code

$ git push heroku master    #Some error occurs here, which cost me few hours to solve.

5. Error

“Permission denied (publickey). fatal: The remote end hung up unexpectedly.”

  check keys:

$ heroku keys

  remove the old keys:

$ heroku keys:remove xxx@xxxxxx

  generate new keys:

$ ssh-keygen -t rsa

$ heroku keys:add /root/.ssh/id_rsa.pub

 

* using ‘sudo’

Web or Native Apps

Although my first product(and few more) in my beta company(means it haven’t been registered yet… ) was accidentally a native app, I still want to implement some of my ideas on web applications.

I don’t mean web apps outweigh native apps. Native app has its advantage, especially the ability to easily use the preinstalled programs like gps, camera, contacts etc. But you have to consider the difference among the platforms, policies etc. Well, to change the subject completely, my first app got on Google Play within minutes, and then approved by Apple four days after submission. I had little expectation  of that, coz Apple had extremely stringent restriction.

All right,  back to original topic. Web application could establish  soon regardless of  multi-platforms. But when developing web apps, we should consider the compatibility with mobile devices.

Something… On the D Day

“I wasted time and now doth time waste me.” 

When I saw this sentence from Richard II of Shakespeare’s on TV, I was about to cry. I have wasted time for few years.
I struggled, confused, despaired… The path of my life was to pursue an academic career, as my friends and family desired. I had no second thought about it, just kept going… until I furthered my study at UK. I found that this path was not for me. I don’t want to head into academic sand despairingly, or wander within a narrow space. I may let my family down, but I would like to choose the way that could stimulate my inner passion.

 

Fortunately, I found something I really want to do few months ago. When I worked hard day and night at restaurant, traveled from north to south on holiday, I got time to think. Some ridiculous ideas just came to my mind, many of which were related to the web world. Meanwhile, I read tech news and subscribed startup mailing list. I could do it on my own, why not?

I am young, while, not that young. I don’t have to achieve something, but I want to do something. Last winter, I took a wonderful trip to North Ireland, where I decided to implement one of my crazy idea (ok, quite normal one) finally.

I took Udacity online course about startup. The core concept was thrilling. Get out of your office, get to know your customer’s pains and gains, and get your product done gradually (MVP – minimum viable product ) in a recurring way. Righto, Let’s rock… As you may know me, I have procrastination. Severe one, I must say. I enjoy the moment shortly before deadline.

 

The first step is most difficult. My grandma encouraged me a lot. Early this month, I began the adventure at last, with my first step  – I had a good chat with my potential client. Yesterday, 16 May, in my opinion, should be remembered as D day – my first ugly but charming product launched at Google Play Store.

I have absolutely no idea about the future. I just want to do something now.

Copy the HTTPS url, then

git clone url    # paste your https url here

ls # check files

cd yourrepositoryname 

subl .  # modified README.md through sublime text

git status # should see “changes not staged for commit”

git add README.md    # git add .         add all files

git status # should see “changes to be commited”

git commit -m “modified the file”    # should see 1 file changed etc.

# if this is the first time, then

git config – global user.name ”your user name”

git config – global user. email your email address
