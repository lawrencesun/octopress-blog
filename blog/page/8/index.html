Macbook Air 11

使用macbook air 11已经快两个月了，已经是我的常用设备，不写进玩转数码显得不够意思。

之前从来没有想过要买苹果的macbook，一来价格昂贵，二来系统不熟悉，没有常用的软件（游戏）。但从去年开始学习编程之后，发现win 系统实在不够友好，因此在raspberry pi 上接触了linux系统。后来为了学习ruby，用virtual box虚拟了ubuntu。期间受到了多篇各路大神关于macbook一步到位的文章的毒害。

因为过去一年里攒了一点钱，加上需要使用keynote, pages等应用，未来开发web app的可能性，于是下狠心买了。英村坑爹的价格也没有阻止我买macbook的决心。后果就是，银行账户基本空了。

为什么买air而不是pro？买11而不是13？

的確，pro的性能更加强劲，屏幕清晰，升级便利。选择air的原因很简单：便携。

Air 11 基本上和我的ipad 2差不多，放在包里不觉得重。不像我的小黑，基本钉死在家里的书桌上了。

今年6月推出的新air续航能力强劲，理论可达9小时。实际使用在6小时左右，应付一天的工作量，绰绰有余。

Mac OS 上手问题

之前完全没有接触过，用过ios和ubuntu多少对mac os的使用提供了些基础。我比较反感买mac装win的人，其实mac os在很多方面都很便捷。触摸板和鼠标操作上手之后就非常方便，以至于我在用小黑时也下意识地用那些手势（当然无效）。虽然以前常见的软件找不到了，破解应用很难找，但从另一个角度来说也是优势，下面会谈到。

Macbook 效率

不少文章都提到过这点，的確，使用macbook，很少会分心。因为没有游戏，没有QQ，没有影音。我给它的定位就是工作机。

祭旗★欧洲申根签证不完全指南

预谋了好几年的欧洲行终于要拉开序幕了。

一大早收到了申根签证，好久没有这么开心了。因为递交材料的时候有几个问题，加上持psw递签的RP论，当时已经做好拒签准备，谁想否极泰来，三天后就收到了这份儿童节大礼。最后给的是一个月多次往返的签。

很少写这种攻略指南类，权当还愿，仅供参考。

所需材料（意签）：

网上很多文章(其实也就是抄来抄去那几篇)详细介绍的，有一些会误导或者失效，因此我强烈建议自己去官网看Document Required。重点说明几点。

1.往返机票。（官网要求fully paid return tickets and  proof of payments must be clearly shown。我的返程机票出了问题，机票确认信上没有支付信息，工作人员要求我补这个材料，我说实在没有，然后就被注明缺少材料，擦。这里强烈鄙视jet2。）

2.国际国内交通。（国内的交通证明是不需要的，但是跨国之间的证明最好有。我用的是interrail的通票，在欧洲住满6个月的人都可以买，26岁一下更便宜，嘿嘿。我是递签的前一天才买，当时怕缺这个有影响。递完之后很后悔，万一拒签，这200多磅就废了。）

3.学校信/雇主信。（在校的好办，但是要注意抬头要写The Consulate General of Italy, London，要有真人签名；我是厚颜无耻地问老板要了抬头纸，然后自己写了假条…嘘…）

4.酒店证明。（用booking.com, hostelworld, ibis 等可以取消预订的都行，但是一定要包含行程的每一天。工作人员是一天接一天查的，不要有gap）

5.资金证明。(银行账单，我的最新账单是到递签前26天的，结果工作人员说不行，要两周以内的，给了我一个小时去补材料。我冒雨前往银行拉了一张没有地址的单，和有地址的单一起交上去了。saving account是不认的。存款当然是越多越好，需要包含你的住宿费加上30磅/天的生活费，工作人员会算出来填到一张表上的。)

6.护照、有效签证(至少还有三个月)、照片、保险。

7.工作人员说是最少七个工作日，实际三个工作日。即便如此，旺季递签还是提早预约的好，万一要缺材料或者被拒，还有机会弥补。

8.上述所谓工作人员都是阿叉…非常的…不说了。

Start Using Heroku and Problem Resolve

1. Signup

2.Download and install Heroku:

toolbelt.heroku.com

Login:

$ heroku login

3.Create heroku remote

$ heroku create

$ git remote -v

$ git remote rm heroku

$ git remote add heroku git@heroku.com:xxxx.git     #Here, xxxx.git is the git file created in the first line of this step

4. Deploying code

$ git push heroku master    #Some error occurs here, which cost me few hours to solve.

5. Error

“Permission denied (publickey). fatal: The remote end hung up unexpectedly.”

  check keys:

$ heroku keys

  remove the old keys:

$ heroku keys:remove xxx@xxxxxx

  generate new keys:

$ ssh-keygen -t rsa

$ heroku keys:add /root/.ssh/id_rsa.pub

 

* using ‘sudo’

** reference:

Although my first product(and few more) in my beta company(means it haven’t been registered yet… ) was accidentally a native app, I still want to implement some of my ideas on web applications.

I don’t mean web apps outweigh native apps. Native app has its advantage, especially the ability to easily use the preinstalled programs like gps, camera, contacts etc. But you have to consider the difference among the platforms, policies etc. Well, to change the subject completely, my first app got on Google Play within minutes, and then approved by Apple four days after submission. All right,  back to original topic. Web application could establish  soon regardless of  multi-platforms. But when developing web apps, we should consider the compatibility with mobile devices.

Ubuntu忘记用户密码怎么办

今天在虚拟ubuntu上创建了ssh之后，一时脑残，把用户改成自动登录，结果sudo时需要密码，怎么输都不对。

大脑再次短路，竟然选择了用百度，“ubuntu忘记用户密码怎么办”。结果99%是一堆误人子弟的烂文，千篇一律，很多文章有“见下图”，你妹图去哪里了，抄袭也要有点节操吧。Google “ubuntu forgot admin password”，立即找到图文详解，而且还介绍了可能会出现的错误。

现在把解决方法记录一下，顺便回馈社会，虽然这篇文章被百度到的机会渺茫。

  1. 首先，不要慌张，重启系统。在开机画面出现之后，狂按左shift进入GRUB界面。（也有说按ESC的，反正我是不行）
  2. 选择recovery mode，进入recovery mode menu。
  3. 选择root     drop to root shell prompt 进入命令行。
  4. 输入passwd user    #user 就是你想要修改密码的用户名，以我为例就是 passwd lawrence
  5. 输入新密码，确认新密码
  6. 输入exit返回menu，选择resume，重启搞定。
  7. 如果出现”Authentication Manipulation error” pla pla 输入“mount -rw -o remount /” 之后再从第4步开始。
自认为说得够详细，图就不配了，因为用skydrive来配图访问越来越慢。

Ubuntu上安装Rails和Sublime Text

基本步骤和上次在raspberry pi上的安装类似，不同的是，安装速度哗哗的。

出现的几个问题都通过stackoverflow上找到答案了。

一个是用rvm安装ruby的时候，出现 

Installing requirements for ubuntu, might require sudo password.
Running 'apt-get --quiet --yes update' would require sudo.
Cowardly refusing to continue, please read 'rvm autolibs'.
Updating repositories
Missing required packages: libgdbm-dev, libffi-dev.
Cowardly refusing to continue, please read 'rvm autolibs'.
解决方法是 
\curl -L https://get.rvm.io | bash -s stable --autolibs=3 --rails
还有一个问题是装完rails之后执行 -v时 出现 
The program 'rails' is currently not installed. You can install it by typing: sudo apt-get install rails
解决方法是

set the ‘Run command as login shell’ checkbox on the Title and Command tab inside of gnome-terminal’s Settings page.

上次在raspberry pi上安装sublime text 2 失败，这次很顺利。 

sudo add-apt-repository ppa:webupd8team/sublime-text-2
sudo apt-get update
sudo apt-get install sublime-text
参考http://wbond.net/sublime_packages/package_control/installation#Testing_Version

安装package control

打开时又出现unable to run package setup…error13的错误

google未果，terminal sudo运行没有问题。

结合错误提示，应该是权限的原因。在ruby-china搜到了解决方法： 

sudo chown -R lawrence:lawrence ~/.config/sublime-text-2
sudo chmod 700 -R ~/.config/sublime-text-2
chown - change file ownership

chmod - modify file access rights

700(rwx——) The file’s owner may read, write, and execute the file. Nobody else has any rights. This setting is useful for programs that only the owner may use and must be kept private from others.
