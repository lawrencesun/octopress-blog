Start Using Heroku and Problem Resolve

1. Signup

2.Download and install Heroku:

toolbelt.heroku.com

Login:

$ heroku login

3.Create heroku remote

$ heroku create

$ git remote -v

$ git remote rm heroku

$ git remote add heroku git@heroku.com:xxxx.git     #Here, xxxx.git is the git file created in the first line of this step

4. Deploying code

$ git push heroku master    #Some error occurs here, which cost me few hours to solve.

5. Error

“Permission denied (publickey). fatal: The remote end hung up unexpectedly.”

  check keys:

$ heroku keys

  remove the old keys:

$ heroku keys:remove xxx@xxxxxx

  generate new keys:

$ ssh-keygen -t rsa

$ heroku keys:add /root/.ssh/id_rsa.pub

 

* using ‘sudo’

** reference:

https://devcenter.heroku.com/articles/keys

http://www.whatibroke.com/?p=284

http://stackoverflow.com/questions/2947190/pushing-app-to-heroku-problem

Web or Native Apps

Although my first product(and few more) in my beta company(means it haven’t been registered yet… ) was accidentally a native app, I still want to implement some of my ideas on web applications.

I don’t mean web apps outweigh native apps. Native app has its advantage, especially the ability to easily use the preinstalled programs like gps, camera, contacts etc. But you have to consider the difference among the platforms, policies etc. Well, to change the subject completely, my first app got on Google Play within minutes, and then approved by Apple four days after submission. I had little expectation  of that, coz Apple had extremely stringent restriction.

All right,  back to original topic. Web application could establish  soon regardless of  multi-platforms. But when developing web apps, we should consider the compatibility with mobile devices.

Something… On the D Day

“I wasted time and now doth time waste me.” 

When I saw this sentence from Richard II of Shakespeare’s on TV, I was about to cry. I have wasted time for few years.
I struggled, confused, despaired… The path of my life was to pursue an academic career, as my friends and family desired. I had no second thought about it, just kept going… until I furthered my study at UK. I found that this path was not for me. I don’t want to head into academic sand despairingly, or wander within a narrow space. I may let my family down, but I would like to choose the way that could stimulate my inner passion.

 

Fortunately, I found something I really want to do few months ago. When I worked hard day and night at restaurant, traveled from north to south on holiday, I got time to think. Some ridiculous ideas just came to my mind, many of which were related to the web world. Meanwhile, I read tech news and subscribed startup mailing list. I could do it on my own, why not?

I am young, while, not that young. I don’t have to achieve something, but I want to do something. Last winter, I took a wonderful trip to North Ireland, where I decided to implement one of my crazy idea (ok, quite normal one) finally.

I took Udacity online course about startup. The core concept was thrilling. Get out of your office, get to know your customer’s pains and gains, and get your product done gradually (MVP – minimum viable product ) in a recurring way. Righto, Let’s rock… As you may know me, I have procrastination. Severe one, I must say. I enjoy the moment shortly before deadline.

 

The first step is most difficult. My grandma encouraged me a lot. Early this month, I began the adventure at last, with my first step  – I had a good chat with my potential client. Yesterday, 16 May, in my opinion, should be remembered as D day – my first ugly but charming product launched at Google Play Store.

I have absolutely no idea about the future. I just want to do something now.

Ubuntu忘记用户密码怎么办

今天在虚拟ubuntu上创建了ssh之后，一时脑残，把用户改成自动登录，结果sudo时需要密码，怎么输都不对。

大脑再次短路，竟然选择了用百度，“ubuntu忘记用户密码怎么办”。结果99%是一堆误人子弟的烂文，千篇一律，很多文章有“见下图”，你妹图去哪里了，抄袭也要有点节操吧。Google “ubuntu forgot admin password”，立即找到图文详解，而且还介绍了可能会出现的错误。

现在把解决方法记录一下，顺便回馈社会，虽然这篇文章被百度到的机会渺茫。

  1. 首先，不要慌张，重启系统。在开机画面出现之后，狂按左shift进入GRUB界面。（也有说按ESC的，反正我是不行）
  2. 选择recovery mode，进入recovery mode menu。
  3. 选择root     drop to root shell prompt 进入命令行。
  4. 输入passwd user    #user 就是你想要修改密码的用户名，以我为例就是 passwd lawrence
  5. 输入新密码，确认新密码
  6. 输入exit返回menu，选择resume，重启搞定。
  7. 如果出现”Authentication Manipulation error” pla pla 输入“mount -rw -o remount /” 之后再从第4步开始。
自认为说得够详细，图就不配了，因为用skydrive来配图访问越来越慢。

ps. 其实应该说是中文搜索和英文搜索，因为google搜中文98%也是出现烂文。

Ubuntu上安装Rails和Sublime Text

基本步骤和上次在raspberry pi上的安装类似，不同的是，安装速度哗哗的。

出现的几个问题都通过stackoverflow上找到答案了。

一个是用rvm安装ruby的时候，出现 

Installing requirements for ubuntu, might require sudo password.
Running 'apt-get --quiet --yes update' would require sudo.
Cowardly refusing to continue, please read 'rvm autolibs'.
Updating repositories
Missing required packages: libgdbm-dev, libffi-dev.
Cowardly refusing to continue, please read 'rvm autolibs'.
解决方法是 
\curl -L https://get.rvm.io | bash -s stable --autolibs=3 --rails
还有一个问题是装完rails之后执行 -v时 出现 
The program 'rails' is currently not installed. You can install it by typing: sudo apt-get install rails
解决方法是

set the ‘Run command as login shell’ checkbox on the Title and Command tab inside of gnome-terminal’s Settings page.

上次在raspberry pi上安装sublime text 2 失败，这次很顺利。 

sudo add-apt-repository ppa:webupd8team/sublime-text-2
sudo apt-get update
sudo apt-get install sublime-text
参考http://wbond.net/sublime_packages/package_control/installation#Testing_Version

安装package control

打开时又出现unable to run package setup…error13的错误

google未果，terminal sudo运行没有问题。

结合错误提示，应该是权限的原因。在ruby-china搜到了解决方法： 

sudo chown -R lawrence:lawrence ~/.config/sublime-text-2
sudo chmod 700 -R ~/.config/sublime-text-2
chown - change file ownership

chmod - modify file access rights

700(rwx——) The file’s owner may read, write, and execute the file. Nobody else has any rights. This setting is useful for programs that only the owner may use and must be kept private from others.

飞行模式下的阅读

飞机是一个世外桃源，刷不了微博，上不了QQ，墙里墙外都开不了花。飞行模式下，只能吃喝拉撒睡，娱乐活动就是读书看电影瞄空姐。

这些年基本都是在旅途中完成阅读，在这个浮躁的时代，花十几个小时看书实在是奢侈。

本来这次想在机场买几本书看，怎料杭州机场的国际候机室居然没有书店。赶紧用机场wifi搜了一本畅销书，李承鹏的《全世界人民都知道》。

这本书比较有名，微博上不少大V推荐（事实证明，如同他们各种烂片推荐，忽悠你没商量），只是我看了一半就看不下去了。

我不太明白为什么这书有关部门还要阻挠签售。书的内容就是博文集合，而且都是些不痛不痒的文章，倒腾几下文字，连讽刺都很牵强，手法单一，很容易就审美疲劳。为了喷而喷，按照大眼自己的说法，就是演得太入戏了。

我在想，要是前些年也投身演艺界，说不定也能跻身右派和公知了。当然我是不会用天天嚷选票这种低端技术的，有选票又如何，能投苍老师吗，大家都是演戏，何必如此较真。那如何取悦和扩大脑残粉呢，大概就是时而愤慨，时而悲壮下吧。

阅读不成，只有看电影了。一翻全是主旋律，林肯，逃离德黑兰… 看得昏昏欲睡。好吧，既来之，则睡之。下次坐飞机还是带上kindle吧。

骑车是一项蛋疼的运动

本来两烩期间，当响应号召吹下牛皮，可惜好久没有扯淡了，所以还是先扯淡吧。

骑车是一项蛋疼的运动，尤其一加速就扯着蛋。我有时会怀疑这是锻炼身体还是折磨身体。当然偶尔骑一下还是不错的。记得小时候为了完成社区实践，常常去参加社区“你丢我捡”活动。骑车有点类似，“你排我吸”，全心全意为人民服雾。“你丢我捡”可以得一个社区主任的盖章，“你排我吸”则更加高尚一些，纯公益活动，连擦鼻灰的纸都自己掏。

我还是很喜欢这项运动的，在骑的时候可以思考人生而不会被队友和球迷骂。以前骑车时胡思乱想，有时骑到一半发现自己身处十字路口，被汽车十面埋伏。

飙车是很有意思的。可惜车烂腿短，没有任何优势，常常被人超车。唯一逆袭的方法，大概就是许愿道路前方有个黑心钉，阻止前方后视灯光的渐渐远逝。在小不列颠，带上头盔手套，开启前后灯，还可以和汽车一起深夜飙车。每当追着警车狂骑的时候，都会让人想起极品飞车热力追踪。

先扯这么多。
