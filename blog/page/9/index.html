去年初上couresa的公开课安装了virtualbox+ubuntu，为了体验最新版本，就把旧版给删了，结果引发了一系列惨案。先是好不容易安装好的12.10显示出错，google之貌似没有一个完美的解决方案，只有降级至12.04，结果又出现“display low quality”的错误，貌似这个问题挺普遍，似乎和A卡驱动有关，神农尝百草般试遍所有解决方案，N次重装之后，终于找到了最佳方案:

飞行模式下的阅读

飞机是一个世外桃源，刷不了微博，上不了QQ，墙里墙外都开不了花。飞行模式下，只能吃喝拉撒睡，娱乐活动就是读书看电影瞄空姐。

这些年基本都是在旅途中完成阅读，在这个浮躁的时代，花十几个小时看书实在是奢侈。

本来这次想在机场买几本书看，怎料杭州机场的国际候机室居然没有书店。赶紧用机场wifi搜了一本畅销书，李承鹏的《全世界人民都知道》。

这本书比较有名，微博上不少大V推荐（事实证明，如同他们各种烂片推荐，忽悠你没商量），只是我看了一半就看不下去了。

我不太明白为什么这书有关部门还要阻挠签售。书的内容就是博文集合，而且都是些不痛不痒的文章，倒腾几下文字，连讽刺都很牵强，手法单一，很容易就审美疲劳。为了喷而喷，按照大眼自己的说法，就是演得太入戏了。

我在想，要是前些年也投身演艺界，说不定也能跻身右派和公知了。当然我是不会用天天嚷选票这种低端技术的，有选票又如何，能投苍老师吗，大家都是演戏，何必如此较真。那如何取悦和扩大脑残粉呢，大概就是时而愤慨，时而悲壮下吧。

阅读不成，只有看电影了。一翻全是主旋律，林肯，逃离德黑兰… 看得昏昏欲睡。好吧，既来之，则睡之。下次坐飞机还是带上kindle吧。

骑车是一项蛋疼的运动

本来两烩期间，当响应号召吹下牛皮，可惜好久没有扯淡了，所以还是先扯淡吧。

骑车是一项蛋疼的运动，尤其一加速就扯着蛋。我有时会怀疑这是锻炼身体还是折磨身体。当然偶尔骑一下还是不错的。记得小时候为了完成社区实践，常常去参加社区“你丢我捡”活动。骑车有点类似，“你排我吸”，全心全意为人民服雾。“你丢我捡”可以得一个社区主任的盖章，“你排我吸”则更加高尚一些，纯公益活动，连擦鼻灰的纸都自己掏。

我还是很喜欢这项运动的，在骑的时候可以思考人生而不会被队友和球迷骂。以前骑车时胡思乱想，有时骑到一半发现自己身处十字路口，被汽车十面埋伏。

飙车是很有意思的。可惜车烂腿短，没有任何优势，常常被人超车。唯一逆袭的方法，大概就是许愿道路前方有个黑心钉，阻止前方后视灯光的渐渐远逝。在小不列颠，带上头盔手套，开启前后灯，还可以和汽车一起深夜飙车。每当追着警车狂骑的时候，都会让人想起极品飞车热力追踪。

先扯这么多。

在raspberry Pi上安装gedit

在尝试安装sublime text 2失败之后，转向了gedit。之前安装了vim，但是作为linux新手，面对vim实在是有点无从下手。

gedit是在尝试安装sublime时装的，至少可以使用。sudo apt-get install gedit

google了一下之后发现很多人推荐安装gmate以达到textmate的效果。

安装方法如下：

sudo apt-add-repository ppa:ubuntu-on-rails/ppa

sudo apt-get update

sudo apt-get install gedit-gmate

可是在安装过程中出现两个错误。第一个错误是没有apt-add-repository指令，似乎可以通过

sudo apt-get install python-software-properties

来解决。参考这里。

但是最终还是安装不了。

其他几个常见的插件可以安装。

sudo apt-get install gedit-plugins

Codecademy, Code School, Treehouse初印象

网络教学或许会掀起一场革命，但是从目前的质量和考核来说，谈替代传统教育还为时过早。

从去年开始，网络教育越办越火，著名的有coursera, udemy, udacity等。论课程数量，coursera的增长最快，合作的大学也越来越多，课程设置和考核基本上是和学校一致的。我之前报了两个班的名，可惜都没能完成。Udacity的一个课程基本上完了，视频的质量很高，讲师也很风趣。对udemy和可汗一类的用户上传型不太喜欢，质量参差不齐。

与此同时，几个编程学习的网站也崛起了，正好准备学，就都去参观了一下。

最早接触的是最有名的Codecademy，当时给我耳目一新的感觉，与传统的教学方法不同，这种网上边学边敲代码的方式效率很高。而类似于游戏打怪的奖励也能激发学习的兴趣。他家的优势是免费，可惜在系统学习方面有点欠缺，就拿我看的ruby来说，有些地方不太清晰，同时内容之间的衔接也不太顺，甚至有跳跃。不过作为入门学习，还是不错的。学习过程中还有问答互动的地方，这点很不错。

Treehouse也是编程学习，需要按月订购，但是在看了课程清单和模式之后，产生的兴趣并不大。一个是课程的完整度还不是很高，再者模式和界面比较沉闷。

Code School是在ruby-china等网站看到介绍去参观的，大多数课程是需要按月订购的。很多人都说可以在一个月内完成课程，因此也是很划算的。他家的教学内容是最容易让人产生兴趣的，比如rails 的僵尸背景，git的星球背景，同时视频、讲义都提供下载，但是没有互动。另外内容偏中等，很多基础没有详细介绍，更多的是一些技巧和重点，好处是很实用，缺点是如果没有基础，看起来会很累，需要不断查资料。

作为提高兴趣和入门讲解，这几个网站都是不错的选择，不过要精益求精还是需要结合文本资料和互动学习。

我个人兴趣和偏好是，Code School>=Codecademy>>Treehouse

在Raspberry Pi上安装Ruby/Rails

Ruby on Rails 目前在国外很火，由于买不起Mac，Raspberry Pi 的Linux环境, 正好用来体验和学习一下Ruby。

装了网卡之后一切安装都变得方便了。

首先当然是Google之了。可是查到的资料不多，而且多数作者也是尝试，没有一个可靠的结论，也不知道运行速度如何。

反正Pi就是用来折腾的，大不了重刷SD卡呗。

#先更新一下

sudo apt-get update

#然后安装Ruby Version Manager（RVM）

curl -L get.rvm.io | bash -s stable –ruby

这里需要等很长时间…有说1小时的，有说3小时的。我决定采纳elinux上的建议，have a sleep and check in the morning.

早上起来Ruby已经装好了~~  need to run ‘source/home/pi/.rvm/scripts/rvm’ in all your open shell window.

ruby -v

#接下来安装rails

gem -v

gem install bundler rails

又要等了。。。好吧，去看会电视剧。。。

让Raspberry Pi 连上 Wifi

Raspberry Pi B型支持网络连接，但是本身并不带有网卡。连不了网很多好玩的东西就无法实现了。

所以立即在Amazon 买了无线网卡，见下图。

由于Pi只有两个USB接口，只能鼠标键盘轮着插入了。。当然键盘更重要一些。

启动后，网卡自动识别了。有两种方法可以联网。

一种就是startx进入桌面操作，点击wifi config， 然后就和windows系统下一样，搜索wifi, 输入密码，done.

还有一种就是命令输入。

sudo nano /etc/network/interfaces

之后会出现默认配置，如下。

修改成以下内容：

auto lo

iface lo inet loopback
iface eth0 inet dhcp

allow-hotplug wlan0       #热插拔
auto wlan0                          #默认使用wlan0设备名
iface wlan0 inet dhcp     #DHCP获取
wpa-ssid “yyy”                   #网络名称
wpa-psk “xxx”                    #密码

修改好了之后

sudo /etc/init.d/networking restart 运行

可以看pi store、可以上网啦！

在pi上学习linux

之前没怎么接触过linux，只在研究生课程computer vision上用过ubuntu。因此在大多数人拿到raspberry pi一脸兴奋的时候，我却登陆和关机都无从下手。

当时从用户手册知道了进入图形界面的指令:

startx

然后等我想关机的时候就傻眼了（好吧, 关电源也行），手册里没有说，只能借助强大的google了。查到关机指令是:

sudo shutdown-h now

当时也没多想。最近开始在codecademy看ruby，自然就看了点linux，方才恍然大悟。原来x就是图形接口 window 的w 下一个字母就是x。

于是暂时搁置用virtualbox安装ubuntu，直接上手用pi来玩一玩吧。

用户登录输入pi，密码是raspberry之后（不可见，当时又吓了我一跳，哎，初学者）出现：

pi@raspberrypi ~ $

pi是用户名，@后面是主机名，~表示在根目录，$是一般用户的提示字符(root用户是#)

好了，下面按照《鸟哥的私房菜》敲几个指令。

pi@raspberrypi ~ $ date   #显示时间

时间显示是Sun Dec 16 23:34:16 GMT 2012 咳 暂时没有联网 结果就是这样了。

还可以改格式，方法是后面加上 +%Y/%m/%d

echo $LANG    #显示语言

LANG=en_US    #修改语言

我当时装的时候选择的语言是 en_GB

在文本模式下的字体实在太小了，暂时没找到方法改变，只好进窗口模式下的terminal敲码了。

cal    # 显示日历 cal 2013 #显示今年的日历啦
